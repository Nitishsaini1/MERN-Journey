📖 Day 11 — My Journal

Date: 14 July

Project: Movie Library v1

Today I completed the first version of my Movie Library project. I didn't just write code—I focused on understanding how JavaScript thinks while executing it.

What I learned

Objects store related information.

Arrays store multiple objects.

push() adds a new object to an array.

for loops let me access every movie using movies[i].

innerHTML = innerHTML + ... first reads the old content, builds a new string, then replaces the old content.

trim() removes spaces only from the beginning and end of a string.

Validation depends on execution order.

Resetting the display before rebuilding prevents duplicate output.


Biggest breakthrough

The biggest breakthrough was realizing that I understand code much better when I replace variables with their actual values and mentally execute JavaScript step by step instead of only reading the syntax.

Biggest challenge

I struggled with:

Understanding object references.

Why displayMovie.innerHTML appears on both sides of the assignment.

Thinking through execution order.

Deciding when to stop improving a project.


How I solved it

I stopped memorizing syntax and started thinking like JavaScript. I learned to ask:

What is the value before this line?

What value is JavaScript creating?

What does memory look like after this line?


Final Result

Movie Library v1 is complete. It validates user input, stores movies in an array of objects, displays the list, and resets the form after adding a movie. This project gave me much more confidence with JavaScript fundamentals.
